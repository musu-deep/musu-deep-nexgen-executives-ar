# NEXGEN EXECUTIVES

## مكتب الرئيس التنفيذي الرقمي

**نظام تشغيل تنفيذي عربي يعمل محليًا من خلال منظومة وكلاء ذكاء اصطناعي متخصصة.**

NEXGEN EXECUTIVES هو مكتب رئيس تنفيذي رقمي يساعد القيادة العليا على إدارة المعرفة المؤسسية، ومتابعة المشروعات والمهام والاجتماعات والقرارات والمخاطر من منصة موحدة.

لا يعمل المشروع بوصفه لوحة معلومات ساكنة، بل بوصفه مكتبًا تنفيذيًا رقميًا نشطًا يستقبل الأحداث، ويفهم السياق، ويوجه العمل إلى الوكيل المختص، ويقترح الإجراء التالي، ثم يراقب التنفيذ ويرفع ما يحتاج إلى قرار أو تصعيد.

---

## أهم تغيير في التشغيل المحلي

أصبح المشروع يعمل افتراضيًا من خلال **باكند مدمج ذاتي الاكتفاء**:

- لا يحتاج إلى تثبيت MongoDB.
- لا يحتاج إلى MongoDB Atlas.
- لا يحتاج إلى تشغيل خدمة قاعدة بيانات منفصلة.
- ينشئ حسابات العرض والبيانات التنفيذية تلقائيًا عند التشغيل.
- يشغّل الواجهة والباكند بأمر واحد.
- يختار منفذًا بديلًا تلقائيًا إذا كان `8001` مستخدمًا.
- يمرر رابط الباكند الصحيح إلى الواجهة، فلا تبقى مرتبطة بمنفذ ثابت.

> قاعدة البيانات المدمجة مناسبة للتشغيل المحلي والتجربة والعروض. عند الانتقال إلى بيئة مؤسسية دائمة يمكن تشغيل مدخل MongoDB الأصلي وربطه بقاعدة مُدارة.

## دورة العمل التنفيذية

1. **الاستقبال:** جمع المشروعات والمهام والاجتماعات والمستندات والرسائل والتحديثات.
2. **الفهم:** تحليل السياق والأولوية والمخاطر والاعتماديات والقرارات المطلوبة.
3. **التوجيه:** إسناد الحالة إلى الوكيل أو المسار التنفيذي المناسب.
4. **التنفيذ:** إنشاء مهام واجتماعات ومتابعات وموجزات وإشعارات قابلة للقياس.
5. **المراقبة:** متابعة المواعيد ونسب الإنجاز والتأخير ومؤشرات الخطر.
6. **التصعيد:** رفع العناصر الحرجة والقرارات المتأخرة إلى الرئيس التنفيذي.
7. **الإحاطة:** إنتاج موجز تنفيذي يومي وصورة لحظية عن حالة المؤسسة.

## منظومة الوكلاء

### مكتب الرئيس التنفيذي

طبقة التنسيق العليا التي تجمع الإشارات التنفيذية، وترتب الأولويات، وتحدد القرارات المطلوبة، وتنتج الإحاطات والتوصيات.

### وكيل ذكاء المشروعات

يربط المشروعات بالمهام والمخاطر والمستندات ونسب الإنجاز، ويكشف التعثر والاعتماديات ونقاط التدخل التنفيذي.

### وكيل ذكاء الاجتماعات

يحوّل الاجتماعات وطلباتها إلى قرارات ومحاضر ومهام ومسؤولين ومواعيد متابعة.

### وكيل مراقبة المخاطر

يرصد المشروعات الحرجة، والمهام المتأخرة، والمؤشرات الحمراء، وبطء اتخاذ القرار، ويقترح مسارات التصعيد.

### وكيل ذكاء المستندات

يستخرج الملخصات والأطراف والتواريخ والالتزامات والبنود المهمة والمخاطر، ويربط المستند بمساره التنفيذي.

### وكيل الاتصالات

ينظم الرسائل والتنبيهات والردود والمتابعات، ويحوّل الاتصال المؤسسي إلى إجراءات قابلة للتتبع.

### وكيل المهام

يتابع المسؤوليات والمواعيد والحالات ونسب الإنجاز، ويكشف المهام العالقة أو التي تحتاج إلى إعادة إسناد.

### وكيل التقارير والإحاطة

ينتج تقارير الأداء والموجز اليومي والقرارات المطلوبة والإجراءات ذات الأولوية والأثر الاستراتيجي.

## البنية التقنية

| الطبقة | التقنية | الدور |
|---|---|---|
| الواجهة | React + Vite + Tailwind CSS | تجربة عربية RTL ولوحات العمل التنفيذية |
| الباكند | FastAPI | واجهات API والمنطق التنفيذي وتنسيق الوكلاء |
| قاعدة التشغيل المحلي | Async Mongo-compatible embedded runtime | تشغيل ذاتي دون خدمة MongoDB خارجية |
| قاعدة الإنتاج الاختيارية | MongoDB | تخزين مؤسسي دائم عند النشر |
| المصادقة | JWT + صلاحيات حسب الأدوار | حماية الوصول وتحديد نطاق الصلاحيات |
| الذكاء الاصطناعي | قواعد محلية + Gemini اختياري | التلخيص والتحليل والإحاطات التنفيذية |

## التشغيل على Windows

بعد تنزيل المشروع أو تحديثه، شغّل الملف التالي بالنقر المزدوج:

```text
start-local.cmd
```

يقوم الملف تلقائيًا بما يلي:

- التحقق من Python وNode.js.
- تثبيت الاعتماديات الناقصة.
- تشغيل الباكند المدمج.
- انتظار جاهزية API.
- تشغيل واجهة Vite بعد نجاح الباكند فقط.
- تمرير منفذ الباكند الصحيح للواجهة.

بعد ظهور رسالة النجاح افتح:

```text
http://localhost:5173
```

## التشغيل من الطرفية

```bash
git pull origin main
npm run dev
```

أو:

```bash
npm run start:local
```

لفحص الباكند:

```text
http://127.0.0.1:8001/api/
```

قد يستخدم النظام منفذًا آخر تلقائيًا إذا كان `8001` مشغولًا؛ يظهر المنفذ الفعلي في نافذة التشغيل.

## تشغيل الباكند منفردًا

### الباكند المدمج — الافتراضي

```bash
npm run start:backend
```

### الباكند المتصل بـMongoDB — اختياري

```bash
npm run start:backend:mongodb
```

وعند استخدام MongoDB أنشئ `backend/.env` واضبط:

```env
MONGO_URL=mongodb://localhost:27017
DB_NAME=ceo_office
JWT_SECRET=replace-with-a-strong-secret
```

## Google Gemini

يعمل النظام دون مفتاح Gemini من خلال القواعد والمخرجات التنفيذية المحلية.

لتفعيل التحليل اللغوي الاختياري أضف في `backend/.env`:

```env
GEMINI_API_KEY=
GEMINI_MODEL=gemini-2.5-flash
```

## النشر

واجهة Vercel لا يجب أن تتصل بـ`localhost`. عند نشر باكند عام أضف في إعدادات Vercel:

```env
VITE_BACKEND_URL=https://your-backend-domain.com
```

أما التشغيل المحلي فلا يحتاج إلى هذا المتغير؛ المشغّل يضبطه تلقائيًا.

# NEXGEN EXECUTIVES

## مكتب الرئيس التنفيذي الرقمي

**نظام تشغيل تنفيذي محلي يعمل من خلال منظومة وكلاء ذكاء اصطناعي متخصصة.**

NEXGEN EXECUTIVES هو مكتب رئيس تنفيذي رقمي يساعد القيادة العليا على إدارة المعرفة المؤسسية، وتنسيق العمل التنفيذي، ومتابعة المشروعات والقرارات والمخاطر والاجتماعات والمهام من منصة عربية موحدة.

لا يعمل النظام بوصفه لوحة معلومات ساكنة، بل بوصفه **مكتبًا تنفيذيًا رقميًا نشطًا** يستقبل البيانات والأحداث، ويفهم السياق، ويوجه العمل إلى الوكيل المتخصص، ويقترح الإجراء التالي، ثم يتابع التنفيذ ويرفع ما يحتاج إلى قرار أو تصعيد.

---

## الفكرة التشغيلية

يحوّل المكتب الرقمي التدفق اليومي للمؤسسة إلى دورة تنفيذية مترابطة:

1. **الاستقبال:** جمع المشروعات والمهام والاجتماعات والمستندات والرسائل والتحديثات.
2. **الفهم:** تحليل السياق والأولوية والمخاطر والاعتماديات والقرارات المطلوبة.
3. **التوجيه:** إسناد كل حالة إلى الوكيل أو المسار التنفيذي المناسب.
4. **التنفيذ:** إنشاء مهام واجتماعات ومتابعات وموجزات وإشعارات قابلة للقياس.
5. **المراقبة:** متابعة المواعيد ونسب الإنجاز والتأخير ومؤشرات الخطر.
6. **التصعيد:** رفع العناصر الحرجة والقرارات المتأخرة إلى الرئيس التنفيذي.
7. **الإحاطة:** إنتاج موجز تنفيذي يومي وصورة لحظية عن حالة المؤسسة.

---

## التشغيل المحلي والخصوصية

صُمم المشروع وفق مبدأ **Local-First**:

- تعمل الواجهة والباكند ومنظومة التنسيق على جهاز المؤسسة أو خادمها الداخلي.
- تحفظ البيانات التشغيلية في MongoDB محلية افتراضيًا.
- تبقى إدارة المستخدمين والمشروعات والمهام والاجتماعات والمستندات داخل بيئة المؤسسة.
- تعمل مسارات الوكلاء وقواعد التوجيه والمتابعة محليًا.
- يمكن تشغيل النظام دون مفتاح لنموذج لغوي؛ عندها يستخدم المخرجات التنفيذية المحلية والقواعد المحددة مسبقًا.
- يمكن تفعيل Google Gemini اختياريًا لتحسين التلخيص والتحليل اللغوي، ولا يُستخدم إلا عند إضافة مفتاح API صراحةً.

> التشغيل المحلي هنا يعني أن التطبيق، والبيانات، والتنسيق بين الوكلاء، وسير العمل التنفيذي تعمل داخل بيئة المؤسسة. أي اتصال بخدمة ذكاء اصطناعي خارجية يظل اختياريًا وتحت تحكم مسؤول النظام.

## منظومة الوكلاء

### مكتب الرئيس التنفيذي

طبقة التنسيق العليا التي تجمع الإشارات التنفيذية، وترتب الأولويات، وتحدد القرارات المطلوبة، وتنتج الإحاطات والتوصيات للرئيس التنفيذي.

### وكيل ذكاء المشروعات

يربط المشروعات بالمهام والمخاطر والمستندات ونسب الإنجاز، ويكشف التعثر والاعتماديات ونقاط التدخل التنفيذي.

### وكيل ذكاء الاجتماعات

يحوّل طلبات الاجتماعات والاجتماعات المجدولة إلى قرارات ومحاضر ومهام ومسؤولين ومواعيد متابعة.

### وكيل مراقبة المخاطر

يرصد المشروعات الحرجة، والمهام المتأخرة، ومؤشرات الأداء الحمراء، وبطء اتخاذ القرار، ويقترح إجراءات التصعيد.

### وكيل ذكاء المستندات

يستخرج الملخصات والأطراف والتواريخ والالتزامات والبنود المهمة والمخاطر، ويربط المستند بالعمل التنفيذي المناسب.

### وكيل الاتصالات

ينظم الرسائل والتنبيهات والردود والمتابعات، ويحوّل الاتصال المؤسسي إلى إجراءات قابلة للتتبع.

### وكيل المهام

يتابع المسؤوليات والمواعيد والحالات ونسب الإنجاز، ويكشف المهام العالقة أو التي تحتاج إلى إعادة إسناد.

### وكيل التقارير

ينتج تقارير الأداء والمحافظ والقطاعات والمشروعات والمهام، ويجهز ملخصات مناسبة للإدارة العليا ومجلس الإدارة.

### وكيل الإحاطة التنفيذية

ينشئ الموجز اليومي، والقرارات المطلوبة، والإجراءات ذات الأولوية، والمخاطر، والأثر الاستراتيجي.

## البنية التقنية

| الطبقة | التقنية | الدور |
|---|---|---|
| الواجهة | React + Vite + Tailwind CSS | تجربة عربية RTL ولوحات العمل التنفيذية |
| الباكند | FastAPI | واجهات API والمنطق التنفيذي والتنسيق بين الوكلاء |
| قاعدة البيانات | MongoDB | الذاكرة المؤسسية والبيانات التشغيلية |
| المصادقة | JWT + صلاحيات مبنية على الأدوار | حماية الوصول وتحديد نطاق الرؤية والإدارة |
| الذكاء الاصطناعي | قواعد محلية + Gemini اختياري | التلخيص والتحليل والإحاطات التنفيذية |
| التشغيل المحلي | Python + Node.js | تشغيل المكتب الرقمي على جهاز أو خادم المؤسسة |

## تشغيل المكتب الرقمي محليًا

شغّل MongoDB أولًا، ثم نفّذ:

```bash
npm run dev
```

يشغّل هذا الأمر:

- الواجهة على `http://localhost:5173`.
- الباكند العربي على `http://localhost:8001`.
- إنشاء `backend/.env` من ملف المثال عند عدم وجوده.
- التحقق من اتصال MongoDB قبل تشغيل النظام.
- إيقاف الواجهة والباكند معًا عند الضغط على `Ctrl+C`.

لفحص الباكند مباشرة:

```text
http://localhost:8001/api/
```

### تشغيل الأجزاء منفردة

```bash
npm run start:backend
npm run dev:frontend
```

---

## تشغيل MongoDB على Windows

افتح PowerShell بصفة مسؤول:

```powershell
Get-Service *mongo*
Start-Service MongoDB
```

ثم أعد تشغيل النظام:

```bash
npm run dev
```

يمكن بدلًا من ذلك وضع رابط MongoDB داخل `backend/.env` في المتغير `MONGO_URL`.

## متغيرات البيئة

### الباكند — `backend/.env`

```env
MONGO_URL=mongodb://localhost:27017
DB_NAME=ceo_office
JWT_SECRET=replace-with-a-strong-secret-at-least-32-characters
COOKIE_SECURE=false
COOKIE_SAMESITE=lax
CORS_ORIGINS=http://localhost:5173,http://127.0.0.1:5173
GEMINI_API_KEY=
GEMINI_MODEL=gemini-2.5-flash
```

ترك `GEMINI_API_KEY` فارغًا يبقي النظام في نمط التشغيل المحلي دون استدعاء Gemini.

### الواجهة — `frontend/.env`

```env
VITE_BACKEND_URL=http://localhost:8001
```

## معالجة خطأ الاتصال بالباكند

ظهور:

```text
localhost:8001/api/... net::ERR_CONNECTION_REFUSED
```

يعني أن الواجهة تعمل بينما الباكند غير شغال. من مجلد المشروع نفّذ:

```bash
npm run dev
```

وتحقق من تشغيل MongoDB إذا ظهرت رسالة تتعلق بقاعدة البيانات.

## النشر المؤسسي

يمكن تشغيل النظام كاملًا على خادم داخلي، أو فصل الواجهة عن الباكند:

- نشر الواجهة على Vercel أو خادم ويب داخلي.
- تشغيل الباكند على خادم Python أو حاوية داخلية.
- تشغيل MongoDB داخل شبكة المؤسسة أو في خدمة مُدارة.

أمر تشغيل الباكند:

```bash
uvicorn backend.arabic_server:app --host 0.0.0.0 --port $PORT
```

وعند نشر الواجهة يجب ضبط:

```env
VITE_BACKEND_URL=https://your-backend-domain.com
```

لا تستخدم `localhost:8001` في نسخة الواجهة المنشورة للعامة؛ يجب أن يشير المتغير إلى عنوان الباكند المتاح لها.
